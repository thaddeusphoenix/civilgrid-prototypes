Paving Data Refresh Status
Purpose

From the 6/22 planning meeting: give program managers a status page showing how far along CivilGrid is on the quarterly paving/moratoria data refresh across the service territory — reachable from where conflicts already live.

What to notice
  • The Data Refresh Status link sits on the Conflicts Summary page (click it to cross over)
  • Top progress tracker is cadence-agnostic ("Cycle 2 of 6") — refresh frequency is per-org, not fixed quarters
  • Per-jurisdiction research status chips, incl. "Done — No Updates" (current, nothing new this cycle)
  • Notes cells are admin-editable (click one and type) — stakeholder ask
  • Opt-in toggle for the monthly summary email that replaces the spreadsheet to the program manager
Build history
  • b004 — dynamic cadence; "Done — No Updates" status; softened button; manual/Linear status note
  • b003 — "Research Status" rename + new states
  • b002 — layout + standard controls
  • b001 — initial build
If we build this — Linear issues
Foundation / data model
  • Derive per-jurisdiction last-refresh date from existing load timestamps
  • Model research status per jurisdiction (Not Started / In Progress / Done — New Data / Done — Changes Received / Done — No Updates / Done — No Data Provided)
  • Per-org refresh cadence config (e.g. a large electric utility ~6×/yr, a smaller gas utility 1×/yr) drives the cycle label + % complete — not fixed calendar quarters
  • Per-jurisdiction admin-editable notes field (static facts + research notes)
  • Compute projects-impacted per jurisdiction via spatial join to project boundaries
Status page UI
  • Add Data Refresh Status route + link from Conflicts Summary
  • Cadence-agnostic progress tracker (cycle N of M) + summary cards
  • Jurisdiction status table with editable notes
Program-manager digest
  • Opt-in monthly/quarterly summary email (jurisdictions updated, projects impacted)
  • Per-org feature flag / service-territory scoping
Decisions & spikes
  • Chunky status first vs. granular research detail (meeting: start chunky)
  • Admin edit surface for notes — inline vs. separate admin view (engineering)
  • Research status likely maintained manually — explore integrating the existing internal Linear project as the source (engineering)
  • Is this org-level (territory) or also project-scoped? (open)
