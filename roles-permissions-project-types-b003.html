Roles & Permissions — Project Type Sub-Setting
Purpose

A customer flagged wanting to stop staff from creating Polygon projects — they think in linear feet, and acres confuse the team. This is a per-role sub-setting under "Create new projects" to restrict which project types a role can create. (Customer Slack thread, 7/22.)

What to notice
  • The new disclosure on "($) Create new projects" nests Linear / Polygon / Circle — the real project geometry types from the New Project flow, not invented names.
  • Per-role sub-rows: a role's type checkboxes grey out automatically when that role's "Create new projects" is off — nothing to restrict by type if a role can't create at all.
  • Org-wide-only was considered (the customer's fallback ask) and ruled out — this stays per-role.
  • Circle is already gated today by an existing org feature flag (circleProjects) — a wrinkle to reconcile before shipping.
  • Expanded by default here so the concept reads immediately in a walkthrough; per the "calm by default" design principle, production would likely default this row collapsed.
Not real yet

No persistence — toggles reset on reload. Nothing else in the app reads this yet; the New Project flow doesn't enforce it.
