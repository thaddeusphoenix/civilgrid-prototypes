Pre-Approved Domains — CivilGrid Admin
Purpose

Clone of the live superadmin screen (/admin/pre-approved-orgs) — lets a CS admin whitelist an email domain so future invited users on that domain skip the manual Persona-approval queue.

Linear project

Improve Org Onboarding & Management & First User Experience (ID Verification) ↗

What to notice
  • Adding a domain requires an explicit confirmation step — the admin attests they've received and filed written proof of the domain's background-check process before the whitelist takes effect
  • Removing a domain doesn't delete the row — it's struck through with a "Removed" date, so the audit trail stays intact
  • This whitelist is a separate mechanism from org auto-join — combining the two would let a user sign up and land in an org with zero manual review, which is why org owners get notified on auto-join (mitigation, not built into this screen)
