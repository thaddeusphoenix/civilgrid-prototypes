Contract Pre-Buy & Usage (A/B)
Purpose

One customer-facing page, two formatting takes, toggled from the Prototype bar. Same data both ways — which layout shows what was bought and how much is used to date fastest? No pacing, no projections, no expected-usage framing.

Linear project

Billing & Fee Transparency ↗

Layout A — tiles + bars
  • 2 stat tiles (used, remaining) + a usage progress bar (drawn vs. remaining)
  • Vertical stacked monthly bar chart — hover to reveal per-tier
  • Scannable, familiar dashboard pattern
Layout B — gauge + trajectory
  • Radial gauge (units used vs. pre-purchased) + summary facts
  • Cumulative units-drawn-to-date line under the pool ceiling — hover the points
  • Monthly detail as horizontal rows with per-tier values shown inline
  • Calm, one-glance status of the contract
What to notice

Toggle A/B in the header. Both show only what was bought and consumption to date — no utilization %, run rate, or projected usage. Both stamp the feedback form with which layout you were viewing.
