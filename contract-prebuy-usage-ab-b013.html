Contract Pre-Buy & Usage
Purpose

The customer-facing view of a contract's pre-purchased project pool and how much has been used. An org can hold multiple contracts — pick one to see its stats and the individual projects that consumed the units. Project count only — no pacing, projections, or dollar figures.

Linear project

Billing & Fee Transparency ↗

What to notice
  • Contract selector (b013). Switch between the org's contracts (each tied to a Salesforce Opportunity) — the usage bar, monthly chart, and projects list all reload per contract.
  • Projects drill-down (b013). The individual projects that make up the used units — name, division, size, units, created. Oversized projects (over 0.5 mi / 20 acres) draw more than one unit; those are flagged.
  • Placement (b013). Surfaced as the org admin's default view, not buried under Settings → Usage (CS: "hidden behind a lot of clicks"). See the nav — Usage is a top-level destination.
  • Single layout (b013). The old A/B is resolved to Layout A (usage bar + monthly chart) — CS and Sales both preferred it. Layout B (gauge + trajectory) is dropped.
  • The pool is denominated in Project Units (project count) — grounded in CivilGrid's real Order Form pricing (Project Unit = 0.5 mi or 20 acres per project). No utilization %, run rate, projected usage, or dollar figures.
Not real yet
