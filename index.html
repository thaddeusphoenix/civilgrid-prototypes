CivilGrid Prototypes

Throwaway concept prototypes for design conversations. Fictional data only.

Paving Trust & Transparency

  • Paving Data Refresh Status Program-manager dashboard showing quarterly paving/moratoria refresh progress across the service territory, linked from the Conflicts Summary.
  • Paving Alert — PM Email Notification email a project manager receives when paving/moratoria data changes within a project boundary.
  • Portfolio Monthly Report — Email Opt-in monthly digest a program manager receives summarizing which jurisdictions were refreshed and how many projects were impacted across the portfolio.
  • Intelligence — Notification Preferences A Notifications sub-tab on the Intelligence tab with a New Paving Data opt-in, mirroring the Utilities pattern.

Large Customer Org Management

  • Account-Level Access (A/B/C) A new top-level Account entity above a customer's orgs — reached from a new "Account Settings" menu item — whose main surface houses access to all projects across every org. Three structures behind one toggle: A — direct per-project grants (projects can be shared across orgs); B — Account → Org Group → Org → Project tree with cascading access; C — attribute rules over Region / Division / Customer / Project Type. Sibling to Member Access Scope.
  • Member Access Scope — Roles & Permissions Org Settings flow for controlling which projects each member can see: an Access tab to define the scope fields (Region / Division / Customer / Project Type), an invite flow that pre-scopes new members, and per-member scope editing on the Members tab.
  • Contract Pre-Buy & Usage Org Settings Usage tab giving an enterprise customer self-serve visibility into their contractual project-area pre-buy — pool consumed vs. remaining, pacing against plan, and monthly draw-down by tier. Toggle A/B in the header for two layout takes.
  • Feature Provisioning — Org Subscription Admin tab for setting which features each project tier includes for an org: locked tier defaults as the baseline, per-org overrides flagged amber with a live count, per-tier on/off switches, and per-org tier descriptions.

Trust & Access Control

  • Pre-Approved Domains — Admin Superadmin screen for whitelisting an email domain so future invited users on that domain skip manual Persona approval — includes the confirmation step attesting background-check proof is on file, and non-destructive remove (struck through, not deleted).
